Проект представляет собой консольное приложение для генерации случайных цитат с возможностью фильтрации по категориям.
Цель проекта — смоделировать процесс командной разработки с использованием Agile и Scrum.
В рамках лабораторной работы используется GitHub Issues для формирования Product Backlog и управления спринтом.
Работа разбита на отдельные инкременты, отражающие функциональные и нефункциональные требования.



Sprint Review:
В течение спринта были успешно завершены задачи, связанные с описанием модели данных и проектированием интерфейса репозитория.
В результате был получен первичный инкремент, формирующий основу для дальнейшей реализации функциональности генерации цитат.
Часть задач, относящихся к пользовательскому интерфейсу и обработке исключительных ситуаций, осталась незавершённой и запланирована на следующую итерацию.

Sprint Retrospective

Что прошло хорошо:
- чёткая постановка задач упростила контроль их выполнения
- использование GitHub Issues позволило наглядно отслеживать прогресс
- приоритизация помогла сфокусироваться на ключевых задачах спринта

Что пошло не так:
- сложность отдельных задач была недооценена
- временные ограничения не позволили завершить весь запланированный объём работ

Что можно улучшить в следующем спринте:
- уточнять оценки сложности на этапе планирования
- делить задачи на более атомарные
- заранее учитывать возможные риски и задержки
